31 class Cancelled(BaseException, metaclass=NoPublicConstructor):
32 """Raised by blocking calls if the surrounding scope has been cancelled.
33  
34 You should let this exception propagate, to be caught by the relevant
35 cancel scope. To remind you of this, it inherits from :exc:`BaseException`
36 instead of :exc:`Exception`, just like :exc:`KeyboardInterrupt` and
37 :exc:`SystemExit` do. This means that if you write something like::
38  
39 try:
40 ...
41 except Exception:
42 ...
43  
44 then this *won't* catch a :exc:`Cancelled` exception.
45  
46 You cannot raise :exc:`Cancelled` yourself. Attempting to do so
47 will produce a :exc:`TypeError`. Use :meth:`cancel_scope.cancel()
48 <trio.CancelScope.cancel>` instead.
49  
50 .. note::
51  
52 In the US it's also common to see this word spelled "canceled", with
53 only one "l". This is a `recent
54 <https://books.google.com/ngrams/graph?content=canceled%2Ccancelled&year_start=1800&year_end=2000&corpus=5&smoothing=3&direct_url=t1%3B%2Ccanceled%3B%2Cc0%3B.t1%3B%2Ccancelled%3B%2Cc0>`__
55 and `US-specific
56 <https://books.google.com/ngrams/graph?content=canceled%2Ccancelled&year_start=1800&year_end=2000&corpus=18&smoothing=3&share=&direct_url=t1%3B%2Ccanceled%3B%2Cc0%3B.t1%3B%2Ccancelled%3B%2Cc0>`__
57 innovation, and even in the US both forms are still commonly used. So
58 for consistency with the rest of the world and with "cancellation"
59 (which always has two "l"s), Trio uses the two "l" spelling
60 everywhere.
61  
62 """

79 class ClosedResourceError(Exception):

81  
82 Note that "closed" here means that *your* code closed the resource,
